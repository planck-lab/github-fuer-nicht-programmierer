Modul 3: Projekte sichtbar machenModule 3: Make projects visible

Modul 3 von 6 · ⏱ ca. 25 MinutenModule 3 of 6 · ⏱ ~25 minutes
Issues als Aufgaben, Labels als Kategorien, Project Boards als Kanban.Issues as tasks, labels as categories, project boards as Kanban.

🎯 Nach diesem Modul kannst du:🎯 After this module you can:
  • 5 Issues mit Titeln, Beschreibungen und Labels erstellenCreate 5 issues with titles, descriptions, and labels
  • Ein Kanban-Board (GitHub Projects) mit 3 Spalten aufsetzenSet up a Kanban board (GitHub Projects) with 3 columns
  • Issues per Drag & Drop zwischen Spalten verschiebenMove issues between columns via drag & drop

Issues sind nicht nur für BugsIssues aren't just for bugs

Ein Issue ist eine universelle Aufgaben-Einheit. Es kann ein Bug sein, aber auch: eine Idee, eine Aufgabe, eine Frage, ein Verbesserungsvorschlag. Denk an Post-Its auf einem Whiteboard — nur digital und nachvollziehbar.An issue is a universal task unit. It can be a bug, but also: an idea, a task, a question, a suggestion. Think of Post-Its on a whiteboard — but digital and traceable.

1
Erstes Issue erstellenCreate your first issue
  1. Öffne dein Repository → klicke auf den Tab „Issues"Open your repository → click the "Issues" tab
  2. Klicke „New issue"Click "New issue"
  3. Titel: Projektplan finalisierenTitle: Finalize project plan
  4. Beschreibung: Schreibe 2-3 Sätze was zu tun istDescription: Write 2-3 sentences about what needs to be done
  5. Klicke „Submit new issue"Click "Submit new issue"
✅ Dein erstes Issue ist da — mit Nummer #1.✅ Your first issue is here — with number #1.
2
Labels vergebenAssign labels
  1. Öffne dein Issue → klicke rechts auf „Labels"Open your issue → click "Labels" on the right
  2. Wähle ein Label (z.B. enhancement oder erstelle ein eigenes: Priorität: hoch)Pick a label (e.g., enhancement or create your own: Priority: high)
💡 Labels sind wie farbige Marker: Sie helfen dir, Issues auf einen Blick zu kategorisieren.💡 Labels are like colored markers: They help you categorize issues at a glance.
✅ Dein Issue hat jetzt ein farbiges Label.✅ Your issue now has a colored label.
3
4 weitere Issues anlegenCreate 4 more issues

Stell dir vor, du organisierst eine Konferenz. Erstelle Issues für:Imagine you're organizing a conference. Create issues for:

  1. Venue recherchieren und buchenResearch and book venue
  2. Speaker anfragenReach out to speakers
  3. Catering organisierenOrganize catering
  4. Teilnehmer-Registrierung aufsetzenSet up attendee registration
✅ Du hast jetzt 5 Issues. Dein Projekt hat Struktur!✅ You now have 5 issues. Your project has structure!
4
Ein Project Board erstellenCreate a project board
  1. Gehe auf github.com → klicke oben auf dein Profilbild → „Your projects"Go to github.com → click your profile picture → "Your projects"
  2. Klicke „New project" → wähle „Board"Click "New project" → choose "Board"
  3. Name: Konferenz-PlanungName: Conference Planning
  4. Du siehst drei Spalten: Todo, In Progress, DoneYou see three columns: Todo, In Progress, Done
✅ Dein Kanban-Board steht!✅ Your Kanban board is ready!
5
Issues ins Board ziehenAdd issues to the board
  1. Im Project Board: Klicke „+ Add item" unten in der Todo-SpalteIn the project board: Click "+ Add item" at the bottom of the Todo column
  2. Suche dein Repository und wähle deine Issues ausSearch your repository and select your issues
  3. Ziehe ein Issue per Drag & Drop von Todo nach In ProgressDrag an issue from Todo to In Progress
💡 Der Unterschied zu Trello/Notion: Issues sind direkt mit deinen Dateien und Commits verknüpft. Wenn du ein Issue schließt, weiß jeder nicht nur DASS es erledigt ist, sondern WIE.💡 The difference from Trello/Notion: Issues are directly linked to your files and commits. When you close an issue, everyone knows not just THAT it's done, but HOW.
✅ Dein Board zeigt den Projektstatus auf einen Blick.✅ Your board shows project status at a glance.

🎉 Modul 3 geschafft!Module 3 complete!

Du hast Issues erstellt, Labels vergeben und ein Kanban-Board aufgesetzt. Dein Projekt ist jetzt sichtbar und organisiert.You created issues, assigned labels, and set up a Kanban board. Your project is now visible and organized.

✅ Erfolgskriterium:✅ Success criterion:
  • Mindestens 5 Issues mit unterschiedlichen LabelsAt least 5 issues with different labels
  • Ein Kanban-Board mit Issues in mindestens 2 verschiedenen SpaltenA Kanban board with issues in at least 2 different columns

📖 Glossar📖 Glossary

Issue
Eine Aufgabe, Idee oder Frage — wie ein digitales Post-It mit Diskussionsmöglichkeit.A task, idea, or question — like a digital Post-It with discussion built in.
Label
Farbiger Marker zum Kategorisieren von Issues (Priorität, Typ, Status).Colored marker for categorizing issues (priority, type, status).
Milestone
Ein Zielpunkt dem du Issues zuordnen kannst — z.B. „Launch v1.0" oder „Konferenz-Tag".A goal point you can assign issues to — e.g., "Launch v1.0" or "Conference day".
Project Board
Kanban-Ansicht (Todo / In Progress / Done) für deine Issues.Kanban view (Todo / In Progress / Done) for your issues.
