Das Problem The Problem

Du kennst das. Drei Leute arbeiten am gleichen Dokument: You know this. Three people working on the same document:

📄 Projektplan.docx
📄 Projektplan_v2.docx
📄 Projektplan_v2_überarbeitet.docx
📄 Projektplan_v3_final.docx
📄 Projektplan_v3_final_FINAL.docx

Niemand weiß, welche Version zählt. Änderungen werden überschrieben. Feedback geht verloren. Nobody knows which version counts. Changes get overwritten. Feedback is lost.

Die Wahrheit The Truth

GitHub ist kein Programmier-Tool. GitHub ist ein Kollaborations- und Versionierungstool. Und der Beweis: GitHub is not a coding tool. GitHub is a collaboration and version control tool. The proof:

Was du hier lernst What you'll learn

📁 Ein Repository ist ein Projektordner mit Gedächtnis📁 A repository is a project folder with memory

Stell dir einen normalen Projektordner vor — aber jede Änderung wird gespeichert. Wer hat was geändert? Wann? Warum? Wie ein Logbuch für dein Projekt. Imagine a normal project folder — but every change is saved. Who changed what? When? Why? Like a logbook for your project.

🔄 Version Control statt Datei-Chaos🔄 Version control instead of file chaos

Eine Datei. Jede Änderung als eigener Schritt gespeichert — mit Zeitstempel, Name und Beschreibung. Das nennt man einen Commit. Du kannst jederzeit zurückspulen. One file. Every change saved as a separate step — with timestamp, name, and description. That's called a commit. You can rewind anytime.

🤝 Zusammenarbeit ohne E-Mail-Ping-Pong🤝 Collaboration without email ping-pong

Feedback direkt an der Stelle. Änderungsvorschläge die man annehmen oder ablehnen kann. Alles transparent, alles nachvollziehbar. Feedback right where it matters. Change suggestions you can accept or reject. Everything transparent, everything traceable.
