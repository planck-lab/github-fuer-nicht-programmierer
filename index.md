# GitHub ist nicht was du denkst.

<!-- GEÄNDERT — Untertitel fokussiert: eine Zielgruppe statt vier -->
### Für alle, die nicht programmieren — aber besser zusammenarbeiten wollen. Kostenlose Lernreise, ganz ohne Code.

---

## Das Problem

Du hast diese Dateien auf deinem Rechner:

```
Projektplan.docx
Projektplan_v2.docx
Projektplan_v2_überarbeitet.docx
Projektplan_v3_final.docx
Projektplan_v3_final_FINAL.docx
Projektplan_v3_final_FINAL_JAN.docx
```

Drei Leute arbeiten am gleichen Dokument. Niemand weiß, welche Version die aktuelle ist. Änderungen werden überschrieben. Feedback geht verloren. Das Chaos wächst mit jedem Tag.

**Und du denkst, GitHub ist nichts für dich, weil du kein Programmierer bist.**

---

## Die Wahrheit

GitHub ist kein Programmier-Tool. GitHub ist ein **Kollaborations- und Versionierungstool**.

Auf GitHub liegen:
- 📜 [Alle deutschen Bundesgesetze](https://github.com/bundestag/gesetze) — als versionierte Textdateien
- 🔬 [COVID-19-Forschungsdaten der Johns Hopkins University](https://github.com/CSSEGISandData/COVID-19) — die zentrale Datenquelle während der Pandemie
- 🏢 [Das komplette Firmenhandbuch von GitLab](https://handbook.gitlab.com/) — 2.000+ Seiten, öffentlich, versioniert
- 🍳 [Kochrezepte](https://github.com/jcallaghan/The-Cookbook) — ja, wirklich

**Keines davon ist Code.**

Wenn du Projekte managst, Texte schreibst, an Dokumenten zusammenarbeitest oder Daten verwaltest — dann ist GitHub für dich.

## Modul 1: GitHub ist nicht was du denkst

### 📖 Was du hier lernst

- Was ein **Repository** ist (Spoiler: ein Projektordner mit Gedächtnis)
- Was **Version Control** bedeutet und warum es das v3_final_FINAL-Problem löst
- **2 echte Beispiele** für GitHub komplett ohne Code
- Warum GitHub ein **Kollaborationstool** ist, kein Programmier-Tool

### Was ist ein Repository?

Stell dir einen ganz normalen Projektordner auf deinem Computer vor. Dokumente, Tabellen, Bilder, Unterordner. So weit, so normal.

Ein **Repository** (kurz: Repo) ist genau so ein Projektordner — aber mit einer Superkraft: **Es hat ein Gedächtnis.**

Jede Änderung wird gespeichert. Nicht nur die aktuelle Version, sondern die komplette Historie. Wer hat was geändert? Wann? Und warum?

Stell dir vor, du arbeitest an einem Projektplan. Dein Kollege ändert eine Deadline. Deine Chefin ergänzt ein Budget. Ein externer Partner fügt Anforderungen hinzu. Bei einem normalen Ordner hast du am Ende Chaos. Bei einem Repository hast du eine saubere Timeline — wie ein **Logbuch für dein Projekt**.

> 📁 Ein Repository = ein Projektordner mit Gedächtnis. Das war's.

### Warum du v3_final_FINAL nie wieder brauchst

Du kennst das: Du arbeitest an einem Dokument und speicherst es ab. Dann nochmal. Und nochmal.

Am Ende hast du: *Projektplan.docx, Projektplan_v2.docx, Projektplan_v2_überarbeitet.docx, Projektplan_v3_final.docx, Projektplan_v3_final_FINAL.docx, Projektplan_v3_final_FINAL_JAN.docx*

**Version Control** löst genau das. Die Idee ist simpel: Du hast **eine** Datei. Immer nur eine. Und jede Änderung wird als eigener Schritt gespeichert — mit Zeitstempel, Name und Beschreibung. Das nennt man einen **Commit**:

- *23. März, 10:15 — Jan: Deadline für Phase 2 auf Juni verschoben*
- *23. März, 14:30 — Lisa: Budget für Marketing ergänzt*
- *24. März, 09:00 — Tom: Anhang mit Risikoanalyse hinzugefügt*

Du kannst jederzeit zurückspulen. Du brauchst nie wieder eine Datei „_FINAL_v7_wirklich" nennen.

### 2 Beispiele die beweisen: GitHub ist nicht nur für Code

🏛️ **Deutsche Bundesgesetze:** Das Repository [bundestag/gesetze](https://github.com/bundestag/gesetze) enthält alle Bundesgesetze als versionierte Textdateien. Du kannst nachvollziehen, wann Paragraphen geändert wurden und was sich geändert hat.

🔬 **COVID-19-Forschungsdaten:** Die [Johns Hopkins COVID-19-Daten](https://github.com/CSSEGISandData/COVID-19) waren während der Pandemie die zentrale Datenquelle für Regierungen und Medien weltweit. Keine Codezeile — nur Tabellen mit Zahlen.

**Überleg mal:** Wo hast du selbst ein Versionsproblem? Wo arbeiten mehrere Leute am gleichen Dokument?
