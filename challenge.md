# 🛠️ Hands-On Challenge: Dein erstes Repository

**Dauer:** ca. 20 Minuten  
**Vorkenntnisse:** Keine. Du brauchst nur einen Browser und eine E-Mail-Adresse.  
**Ergebnis:** Am Ende hast du ein öffentliches Repository mit einer Startseite und einer Datei — live im Internet.

[← Zurück zur Übersicht](./)

---

## Schritt 1: GitHub-Account anlegen (5 Min)

1. Öffne **[github.com](https://github.com)** in deinem Browser.
2. Klicke oben rechts auf **„Sign up"**.
3. Gib deine **E-Mail-Adresse** ein.
4. Wähle ein **Passwort**.
5. Wähle einen **Benutzernamen** — das wird Teil deiner GitHub-URL.
   - Nimm deinen echten Namen oder eine professionelle Variante (z.B. `jan-rummel`, `lisa-schmidt`)
   - Keine Sonderzeichen außer Bindestrichen
   - Kurz und merkbar
6. Bestätige das **Captcha**.
7. Klicke **„Create account"**.
8. Gib den **Bestätigungscode** aus deiner E-Mail ein.
9. Personalisierung? → **„Skip personalization"**

### ✅ Checkpoint
Du bist eingeloggt und siehst dein GitHub-Dashboard. Du hast einen GitHub-Account!

---

## Schritt 2: Erstes Repository erstellen (3 Min)

1. Klicke auf das **„+"** oben rechts neben deinem Profilbild.
2. Wähle **„New repository"**.
3. Fülle das Formular aus:

| Feld | Eingabe |
|------|---------|
| **Repository name** | `mein-erstes-projekt` |
| **Description** | `Mein erstes GitHub-Repository — ein Experiment ohne Code.` |
| **Public / Private** | **Public** auswählen |
| **Add a README file** | ✅ **Häkchen setzen!** |

4. Klicke **„Create repository"**.

### ✅ Checkpoint
Du siehst dein Repository mit einer Datei: `README.md`.

---

## Schritt 3: README.md bearbeiten (5 Min)

Die README.md ist die Startseite deines Projekts. Jeder, der dein Repository besucht, sieht sie als erstes.

1. Klicke auf **`README.md`** in der Dateiliste.
2. Klicke auf das **Stift-Icon** ✏️ (oben rechts, „Edit this file").
3. **Lösche den Inhalt** und kopiere dieses Template hinein:

```markdown
# Mein erstes Projekt

## Worum geht es?
Dieses Repository ist mein erster Gehversuch auf GitHub.
Ich bin kein Programmierer — ich nutze GitHub als 
Kollaborations- und Organisationstool.

## Was ist hier drin?
- 📄 Diese README als Startseite
- 📁 Dokumente und Notizen zu meinem Projekt
- 📝 Alles versioniert — jede Änderung wird gespeichert

## Über mich
Ich bin [DEIN NAME] und arbeite als [DEIN BERUF/ROLLE].
Dieses Repo ist Teil der Lernreise 
"GitHub für Nicht-Programmierer".

## Was ich gelernt habe
- Ein Repository ist ein Projektordner mit Gedächtnis
- Version Control löst das "v3_final_FINAL"-Problem
- GitHub ist nicht nur für Code

---
*Erstellt am [DATUM] als Teil von Modul 1.*
```

4. Ersetze die **Platzhalter** mit deinen echten Daten.
5. Klicke **„Commit changes..."**
6. Im Popup:
   - **Commit message:** `README mit persönlichen Infos ergänzt`
   - Wähle **„Commit directly to the main branch"**
   - Klicke **„Commit changes"**

### ✅ Checkpoint
Deine README wird formatiert angezeigt — mit Überschriften, Listen und Emojis. **Das ist dein erster Commit!**

---

## Schritt 4: Eine Datei hochladen (3 Min)

1. Gehe zur **Hauptseite deines Repositories**.
2. Klicke **„Add file"** → **„Upload files"**.
3. Ziehe eine Datei in das Upload-Feld. Zum Beispiel:
   - Ein Foto 📸
   - Eine PDF-Datei 📄
   - Eine Excel-Tabelle 📊
   - Oder erstelle eine `notizen.txt` mit ein paar Zeilen Text
4. **Commit message:** `Erste Datei hochgeladen`
5. Klicke **„Commit changes"**

### ✅ Checkpoint
In der Dateiliste siehst du jetzt `README.md` und deine hochgeladene Datei.

---

## Schritt 5: Ergebnis bewundern (2 Min)

1. Kopiere die **URL** aus deiner Adresszeile:
   ```
   https://github.com/dein-benutzername/mein-erstes-projekt
   ```
2. Diese URL kannst du mit jedem teilen!
3. Klicke auf **„2 commits"** über der Dateiliste — das ist die **Commit-Historie**, das Gedächtnis deines Projektordners.

---

## 🎉 Geschafft!

**Dein erstes Repository ist live.** Du hast:

- ✅ Einen GitHub-Account angelegt
- ✅ Ein Repository erstellt
- ✅ Eine README.md geschrieben
- ✅ Eine Datei hochgeladen
- ✅ Zwei Commits gemacht
- ✅ Deine Commit-Historie angeschaut

Alles im Browser. Kein Code. Kein Terminal.

---

## Bonus (für Neugierige)

**Ordner erstellen:** Klicke „Add file" → „Create new file". Gib als Dateinamen `notizen/ideen.md` ein — der Schrägstrich erstellt automatisch einen Ordner!

**Fremdes Repo erkunden:** Besuche [github.com/bundestag/gesetze](https://github.com/bundestag/gesetze) und klicke dich durch die Commit-Historie.

**Stern vergeben:** Wenn dir ein Repo gefällt, klicke auf ⭐ „Star" — wie ein Lesezeichen + Daumen hoch.
